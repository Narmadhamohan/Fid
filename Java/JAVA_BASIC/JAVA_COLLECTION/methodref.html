Method Reference

Method Reference in Java

Method reference is a shorthand notation of a lambda expression to call a method directly by referring to it with a double colon :: syntax.

Types of method references:

  • Reference to a static method: ClassName::staticMethodName
  • Reference to an instance method of a particular object: object::instanceMethodName
  • Reference to an instance method of an arbitrary object of a particular type: ClassName::instanceMethodName
  • Reference to a constructor: ClassName::new
Example: 

// Static method reference
class MethodReferenceExample {
    public static void main(String[] args) {
        // Lambda expression
        Runnable r1 = () -> MethodReferenceExample.sayHello();
        
        // Method reference
        Runnable r2 = MethodReferenceExample::sayHello;

        r1.run();
        r2.run();
    }

    public static void sayHello() {
        System.out.println("Hello from method reference!");
    }
}
Lambda vs Method Reference

When to Choose Lambda vs Method Reference

In Java, both Lambda expressions and Method references are ways to simplify code, but choosing between the two depends on the specific situation:

  • Use Lambda Expression when:
    • You need to perform custom logic inside the lambda.
    • The logic involves more than just invoking a method. For example, you may need to combine multiple actions inside the lambda.
    • It improves readability, especially for inline or short functions.
  • Use Method Reference when:
    • The lambda expression is simply calling an existing method.
    • It makes the code cleaner and more concise because the method reference eliminates boilerplate code.
    • You are dealing with a static method, instance method, or constructor, and want a quick reference.
Example:
  • Using Lambda:
    • 
List names = Arrays.asList("John", "Jane", "Doe");
names.forEach(name -> System.out.println(name)); // Lambda Expression
  • Using Method Reference:
    • 
names.forEach(System.out::println); // Method Reference
  • Example of a Lambda involving multiple actions:
    • 
List names = Arrays.asList("John", "Jane", "Doe");

names.forEach(name -> {
    System.out.println("Processing: " + name);
    String uppercaseName = name.toUpperCase();
    System.out.println("Uppercased: " + uppercaseName);
    // More logic can go here
});

    Explanation: In this case, the lambda expression is preferred because it allows us to do multiple things such as printing, modifying the name, and any other logic. A method reference is less suitable here because it can only refer to a single method, not multiple actions or steps.
