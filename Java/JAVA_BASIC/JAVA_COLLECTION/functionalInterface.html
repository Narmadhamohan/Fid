Functional Interfaces in Java

Overview

A Functional Interface is an interface that contains exactly one abstract method. These interfaces can be represented with lambda expressions, enabling a functional programming style in Java.

Significance of Functional Interfaces

Functional interfaces allow you to pass behavior as parameters to methods, making your code more flexible and reusable. They play a crucial role in enabling features such as the Stream API and event handling.

Example of a Functional Interface

    // Defining a functional interface
    @FunctionalInterface
    interface MyFunctionalInterface {
        void performAction(String message);
    }

    // Using a lambda expression
    MyFunctionalInterface action = (msg) -> System.out.println(msg);
    action.performAction("Hello, Functional Interface!");

Common Built-in Functional Interfaces

Java provides several built-in functional interfaces in the java.util.function package:

Functional Interface Description Example Usage
Supplier<T> Represents a supplier of results. 
Supplier<String> supplier = () -> "Hello";
Consumer<T> Represents an operation that accepts a single input argument and returns no result. 
Consumer<String> consumer = msg -> System.out.println(msg);
Function<T, R> Represents a function that accepts one argument and produces a result. 
Function<String, Integer> lengthFunction = str -> str.length();
Predicate<T> Represents a predicate (boolean-valued function) of one argument. 
Predicate<Integer> isEven = num -> num % 2 == 0;

Conclusion

Understanding functional interfaces is essential for leveraging lambda expressions and functional programming features in Java. They provide a powerful way to create reusable code and improve the expressiveness of your applications.
