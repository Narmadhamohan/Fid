Java Exception Handling Concepts

  • 1. Built-in Exceptions Built-in exceptions are the exceptions that are available in Java libraries. These exceptions are suitable to explain certain error situations.
  • Checked Exceptions: Checked exceptions are called compile-time exceptions because these exceptions are checked at compile-time by the compiler.
  • Unchecked Exceptions: The unchecked exceptions are just opposite to the checked exceptions.
    The compiler will not check these exceptions at compile time. In simple words, if a program throws an unchecked exception, and even if we didnâ€™t handle or declare it, the program would not give a compilation error.
    1. Error: An Error indicates a serious problem that a reasonable application should not try to catch.
    2. Exception: Exception indicates conditions that a reasonable application might try to catch.

Exception keywords

  • Throwing Exceptions: Use the throw keyword to create and signal an error, e.g., throw new IllegalArgumentException("Invalid argument!");.
  • throws: Indicates a method can throw exceptions, allowing callers to handle them. Example: public void myMethod() throws IOException { ... }.
  • try-catch: A mechanism to handle exceptions without crashing the program. Example: 
    try { ... } catch (ExceptionType e) { ... }
  • Logging Error Messages: Records errors for debugging, e.g., logger.severe("An error occurred: " + e.getMessage());.
