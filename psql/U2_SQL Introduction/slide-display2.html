1. Introduction to SQL

SQL (Structured Query Language) is a standardized language used to manage relational databases. It allows for querying, updating, and managing data, as well as controlling access to it. SQL is widely used in applications that involve database management systems like MySQL, PostgreSQL, and SQL Server.

  • Queries: Fetch and manipulate data stored in relational databases.
  • Commands: SQL commands like SELECT, INSERT, UPDATE, DELETE, and more.
  • Transactions: SQL supports transactions that allow multiple operations to be treated as a single unit of work.

2. DDL, DML, DCL, TCL Commands

SQL commands are categorized into different groups based on their functionality:

  • DDL (Data Definition Language): Defines database schema and structure.
    • CREATE: Create a new table or database.
    • ALTER: Modify an existing table.
    • DROP: Delete a table or database.
    • TRUNCATE: Remove all records from a table, without logging individual row deletions.
  • DML (Data Manipulation Language): Manages data within tables.
    • INSERT: Insert new data into a table.
    • UPDATE: Update existing data in a table.
    • DELETE: Remove data from a table.
    • SELECT: Retrieve data from a table based on specific conditions.
  • DCL (Data Control Language): Controls access to the data.
    • GRANT: Grants access rights.
    • REVOKE: Removes access rights.
  • TCL (Transaction Control Language): Manages transactions within the database.
    • COMMIT: Save changes made in a transaction.
    • ROLLBACK: Undo changes in a transaction.
    • SAVEPOINT: Set a savepoint within a transaction.

3. Datatypes and Its Usage in SQL (PostgreSQL)

Datatypes define the type of data that can be stored in a table's columns. In PostgreSQL, the datatype ensures proper handling and optimization of data. Here are the key PostgreSQL datatypes:

  • Numeric Data Types:
    • SMALLINT: Small-range integer (-32,768 to 32,767).
    • INTEGER: Standard integer (-2,147,483,648 to 2,147,483,647).
    • BIGINT: Large-range integer (-9,223,372,036,854,775,808 to 9,223,372,036,854,775,807).
    • DECIMAL(p, s): Exact numeric type with precision p and scale s (e.g., DECIMAL(10, 2)).
    • NUMERIC(p, s): Similar to DECIMAL, used for exact numeric types.
    • REAL: Single precision floating-point number (4 bytes).
    • DOUBLE PRECISION: Double precision floating-point number (8 bytes).
    • SERIAL: Auto-incrementing integer.
  • Character/String Data Types:
    • CHAR(n): Fixed-length string.
    • VARCHAR(n): Variable-length string.
    • TEXT: Variable-length string with no length limit.
  • Date/Time Data Types:
    • DATE: Stores a date (e.g., '2024-10-07').
    • TIMESTAMP: Stores both date and time (without time zone).
    • TIMESTAMPTZ: Stores both date and time (with time zone).
    • TIME: Stores time of day (without date).
    • INTERVAL: Represents time span.
  • Boolean Data Type:
    • BOOLEAN: Stores TRUE or FALSE values.
  • UUID Data Type:
    • UUID: Stores universally unique identifiers.
  • Array Data Type:
    • ARRAY: Allows storage of arrays (e.g., INTEGER[], TEXT[]).
  • JSON Data Types:
    • JSON: Stores JSON data (without validation).
    • JSONB: Stores JSON data (with validation and indexing).
  • Geometric Data Types:
    • POINT, LINE, CIRCLE: Represent geometric shapes like points and circles.
  • Other Data Types:
    • BYTEA: Stores binary data.
    • ENUM: Stores a static, ordered set of values (e.g., 'low', 'medium', 'high').
    • XML: Stores XML data.

4. SQL Operators

SQL Operators are used to perform operations on data. They are essential for filtering results, performing calculations, and managing conditions in queries.

  • Arithmetic Operators: Used for calculations.
    • +: Addition.
    • -: Subtraction.
    • *: Multiplication.
    • /: Division.
  • Comparison Operators: Used to compare values.
    • =: Equal to.
    • != or <>: Not equal to.
    • >: Greater than.
    • <: Less than.
    • >=: Greater than or equal to.
  • Logical Operators: Used to combine conditions.
    • AND: Returns true if both conditions are true.
    • OR: Returns true if any condition is true.
    • NOT: Negates a condition.

Example SQL Query Using SELECT and Operators

SELECT FirstName, LastName
FROM Employees
WHERE Age >= 30 AND Salary > 50000;

Example TRUNCATE Statement

TRUNCATE TABLE Employees;
