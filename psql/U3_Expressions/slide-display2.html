1. Operators Used in SQL

SQL operators are used to perform various operations on data. Here are some key categories of SQL operators:

  • Arithmetic Operators: Perform mathematical calculations.
    • +: Addition (e.g., SELECT 5 + 2;)
    • -: Subtraction (e.g., SELECT 10 - 3;)
    • *: Multiplication (e.g., SELECT 4 * 5;)
    • /: Division (e.g., SELECT 20 / 4;)
  • Comparison Operators: Compare values.
    • =: Equal to.
    • <> or !=: Not equal to.
    • >: Greater than.
    • <: Less than.
    • >=: Greater than or equal to.
    • <=: Less than or equal to.
  • Logical Operators: Used to combine conditions.
    • AND: Returns true if both conditions are true.
    • OR: Returns true if either condition is true.
    • NOT: Negates a condition.
  • Other Operators:
    • BETWEEN: Check if a value is within a range (e.g., age BETWEEN 20 AND 30).
    • IN: Check if a value matches any value in a list (e.g., country IN ('USA', 'Canada')).
    • LIKE: Search for a pattern (e.g., name LIKE 'A%').
    • IS NULL: Check if a value is null.

2. Select Clause and Condition

The SELECT clause is used to query specific data from a database. It retrieves data from one or more tables and is usually followed by conditions.

  • Syntax: SELECT column1, column2 FROM table_name [WHERE condition];
  • Example:
    • SELECT FirstName, LastName
FROM Employees
WHERE Age > 30;
  • Aliasing: You can assign a temporary name to a column using the AS keyword.
    • SELECT FirstName AS "First Name", LastName AS "Last Name"
FROM Employees;

3. Usage of WHERE and HAVING Clauses

WHERE and HAVING are used to filter results in SQL:

  • WHERE Clause: Filters rows before grouping.
    • Syntax: SELECT * FROM table_name WHERE condition;
    • Example:
      • SELECT * FROM Employees
WHERE Age > 30;
  • HAVING Clause: Filters groups after grouping.
    • Syntax: SELECT column, AGGREGATE_FUNCTION(column) FROM table_name GROUP BY column HAVING condition;
    • Example:
      • SELECT Department, COUNT(*)
FROM Employees
GROUP BY Department
HAVING COUNT(*) > 10;

4. Different Types of Operations in SQL

SQL supports various operations on data:

  • Set Operations:
    • UNION: Combine results of two queries, removing duplicates.
    • UNION ALL: Combine results of two queries, keeping duplicates.
    • INTERSECT: Returns rows common to both queries.
    • EXCEPT: Returns rows from the first query that are not in the second query.
  • Join Operations:
    • INNER JOIN: Returns records that have matching values in both tables.
    • LEFT JOIN: Returns all records from the left table and the matched records from the right table.
    • RIGHT JOIN: Returns all records from the right table and the matched records from the left table.
    • FULL OUTER JOIN: Returns all records when there is a match in either left or right table.

5. Usage of Group Function in SELECT Command

Group functions, also called aggregate functions, operate on sets of rows to give one result per group:

  • COUNT(): Returns the number of rows.
  • SUM(): Returns the sum of a numeric column.
  • AVG(): Returns the average value of a numeric column.
  • MAX(): Returns the maximum value.
  • MIN(): Returns the minimum value.

Example Group Function Usage

SELECT Department, COUNT(*), AVG(Salary)
FROM Employees
GROUP BY Department;
