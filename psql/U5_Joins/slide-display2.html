1. Set Operators in SQL

SQL Set Operators are used to combine the results of two or more SELECT queries. The rows returned by the SELECT statements are combined into a single result set. Set operators include:

  • UNION: Combines the results of two SELECT queries, removing duplicate rows.
    • SELECT Name FROM Employees
UNION
SELECT Name FROM Managers;
  • UNION ALL: Combines results of two SELECT queries but keeps duplicate rows.
    • SELECT Name FROM Employees
UNION ALL
SELECT Name FROM Managers;
  • INTERSECT: Returns only the rows that are common to both SELECT queries.
    • SELECT Name FROM Employees
INTERSECT
SELECT Name FROM Managers;
  • EXCEPT: Returns rows from the first query that are not in the second query.
    • SELECT Name FROM Employees
EXCEPT
SELECT Name FROM Managers;

2. Introduction to Joins

SQL Joins are used to combine rows from two or more tables based on a related column. The most common types of joins are:

  • INNER JOIN: Returns records that have matching values in both tables.
  • LEFT JOIN: Returns all records from the left table, and the matched records from the right table.
  • RIGHT JOIN: Returns all records from the right table, and the matched records from the left table.
  • FULL OUTER JOIN: Returns all records when there is a match in either left or right table.

3. Simple Join

A Simple Join (or Inner Join) is the most basic type of join, where it returns rows when there is a match in both tables being joined.

  • Syntax: SELECT columns FROM table1 JOIN table2 ON table1.column = table2.column;
  • Example:
    • SELECT Employees.Name, Departments.DeptName
FROM Employees
INNER JOIN Departments
ON Employees.DeptID = Departments.DeptID;

4. Equi Join

An Equi Join is a specific type of join where the join condition uses the equal operator (=) to match columns.

  • Syntax: SELECT columns FROM table1 JOIN table2 ON table1.column = table2.column;
  • Example:
    • SELECT Orders.OrderID, Customers.CustomerName
FROM Orders
INNER JOIN Customers
ON Orders.CustomerID = Customers.CustomerID;

5. Non-Equi Join

A Non-Equi Join is a join condition that uses operators other than equal (=) to compare columns, such as >, <, or >=.

  • Syntax: SELECT columns FROM table1 JOIN table2 ON table1.column <> table2.column;
  • Example:
    • SELECT Employees.Name, Salaries.Salary
FROM Employees
INNER JOIN Salaries
ON Employees.SalaryRange >= Salaries.MinSalary
AND Employees.SalaryRange <= Salaries.MaxSalary;

6. Self Join

A Self Join is a join in which a table is joined with itself. It is useful when you need to compare rows within the same table.

  • Syntax: SELECT A.column, B.column FROM table A, table B WHERE condition;
  • Example:
    • SELECT A.EmployeeID, A.Name, B.Name AS Manager
FROM Employees A, Employees B
WHERE A.ManagerID = B.EmployeeID;

7. Cartesian Product

A Cartesian Product occurs when there is no specified join condition between two tables, resulting in each row from the first table being combined with every row from the second table.

  • Syntax: SELECT columns FROM table1, table2;
  • Example:
    • SELECT Employees.Name, Departments.DeptName
FROM Employees, Departments;
  • Note: Cartesian products can result in large result sets and should generally be avoided unless explicitly needed.
