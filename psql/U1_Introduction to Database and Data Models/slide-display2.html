1. Database Introduction

A Database is a structured collection of data that is organized so it can easily be accessed, managed, and updated. Databases are used to store large amounts of information efficiently and are vital in applications like banking, retail, and social media.

  • DBMS (Database Management System): Software that manages databases, allowing users to create, read, update, and delete data efficiently.
  • Types of Databases: Relational, NoSQL, Object-Oriented, Distributed Databases, etc.

2. Types of Data Models

Data models are conceptual frameworks that define how data is stored, accessed, and related to other data.

  • Hierarchical Data Model: Data is organized in a tree-like structure where a single parent can have many children.
  • Network Data Model: Allows more complex relationships by permitting each child to have multiple parents.
  • Relational Data Model: Data is organized into tables (or relations), where each table is a set of rows and columns.
  • Object-Oriented Data Model: Data is represented as objects, similar to OOP principles.

3. Introduction to RDBMS

Relational Database Management Systems (RDBMS) store data in tables where relationships between data are defined using keys (primary, foreign). Popular RDBMS software includes:

  • MySQL
  • PostgreSQL
  • Oracle
  • SQL Server

RDBMSs follow ACID properties (Atomicity, Consistency, Isolation, Durability) to ensure reliability.

4. Codd's Rules

Dr. Edgar F. Codd proposed 12 rules that define the requirements for a database management system to be considered relational:

  • Rule 1: The information rule – All information should be stored in tables.
  • Rule 2: Guaranteed access – Each item should be accessible using a combination of table name, primary key, and column.
  • Rule 3: Systematic treatment of null values – Null values must be uniformly treated as unknown or absent.

5. Normalization

Normalization is the process of organizing data in a database to reduce redundancy and improve data integrity. The steps include:

  • 1NF (First Normal Form): Remove duplicate columns and ensure each column contains atomic values.
  • 2NF (Second Normal Form): Achieved when it is in 1NF and all non-key attributes are fully dependent on the primary key.
  • 3NF (Third Normal Form): Achieved when it is in 2NF and there are no transitive dependencies.

6. ER-Diagrams (Entity-Relationship Diagrams)

An ER Diagram is a visual representation of the entities in a database and the relationships between them.

  • Entities: Represent objects or things (e.g., User, Product).
  • Attributes: Characteristics of entities (e.g., User Name, Product ID).
  • Relationships: Represent associations between entities (e.g., A User can place an Order).

Example ER Diagram

        Entity User
            ├── User ID
            ├── User Name
            ├── Email
        Entity Order
            ├── Order ID
            ├── Order Date
            └── Amount
        Relationship: A User can place multiple Orders.

Example SQL Query

Here's a simple SQL query to fetch users who placed orders:

SELECT u.UserName, o.OrderID, o.OrderDate
FROM Users u
JOIN Orders o ON u.UserID = o.UserID
WHERE o.OrderDate > '2023-01-01';
