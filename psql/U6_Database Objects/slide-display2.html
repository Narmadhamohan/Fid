1. Sequence

A Sequence in SQL is a database object that generates a sequence of numeric values according to a specified increment. It is often used to auto-generate primary key values.

  • Syntax: CREATE SEQUENCE sequence_name START WITH start_value INCREMENT BY increment_value;
  • Example:
    • CREATE SEQUENCE seq_employee_id
START WITH 1
INCREMENT BY 1;
  • Explanation: This sequence starts at 1 and increments by 1 each time a new value is requested.
  • Usage: INSERT INTO Employees (EmployeeID, Name) VALUES (seq_employee_id.NEXTVAL, 'John');

2. Synonym

A Synonym in SQL is an alias for another database object, such as a table, view, or sequence. It simplifies the referencing of complex object names, particularly for large applications.

  • Syntax: CREATE SYNONYM synonym_name FOR object_name;
  • Example:
    • CREATE SYNONYM emp FOR Employees;
  • Usage: After creating a synonym, you can use emp as a shorthand for referencing the Employees table: SELECT * FROM emp;

3. View

A View is a virtual table in SQL that is based on the result set of a `SELECT` query. It does not store data itself but provides a way to represent data from one or more tables.

  • Syntax: CREATE VIEW view_name AS SELECT columns FROM table WHERE condition;
  • Example:
    • CREATE VIEW emp_view AS
SELECT EmployeeID, Name, Salary
FROM Employees
WHERE Salary > 50000;
  • Explanation: The view emp_view provides a simplified representation of employees who earn more than 50,000.
  • Operations on View:
    • Querying: SELECT * FROM emp_view;
    • Updating: Views can often be updated if they reference a single base table.
      • UPDATE emp_view
SET Salary = 60000
WHERE EmployeeID = 1;
    • Dropping: DROP VIEW emp_view;

4. Indexing and Types

Indexing improves the speed of data retrieval operations in a database by creating an index on one or more columns. It is similar to the index in a book.

  • Types of Indexes:
    • Unique Index: Ensures that all values in the indexed column are unique.
      • CREATE UNIQUE INDEX idx_employee_id ON Employees(EmployeeID);
    • Non-Unique Index: Allows duplicate values in the indexed column.
      • CREATE INDEX idx_name ON Employees(Name);
    • Composite Index: Indexes multiple columns together to improve query performance.
      • CREATE INDEX idx_name_dept ON Employees(Name, Department);
  • Operations on Index:
    • Creating Index: CREATE INDEX index_name ON table_name(column_name);
    • Dropping Index: DROP INDEX index_name;

5. Clusters

Clusters are a method of storing related tables together in the same database block. They are useful when tables are frequently joined in queries. By clustering tables, you can reduce I/O operations and improve performance.

  • Syntax: CREATE CLUSTER cluster_name (column_name datatype);
  • Example:
    • CREATE CLUSTER emp_dept_cluster (DeptID INT);
  • Explanation: This creates a cluster for tables that share the DeptID column. Related rows from the `Employees` and `Departments` tables will be stored together.
  • Adding Tables to Cluster: You can add tables to the cluster by specifying the cluster name when creating the table.
    • CREATE TABLE Employees (
    EmployeeID INT,
    DeptID INT,
    Name VARCHAR(50)
) CLUSTER emp_dept_cluster (DeptID);
