Introduction to Stacks

  • Definition: A stack is a data structure following Last In First Out (LIFO).
  • Example: Like stacking plates; last added is the first removed.
  • Usage: Operations only happen at the top of the stack.
  • Tip: Think of a stack like a stack of books – last book placed is the first one taken.

Stack Implementation

  • Push: Adds an element to the top of the stack.
  • Pop: Removes the element from the top of the stack.
  • Peek: Views the top element without removing it.
  • Overflow: Happens when the stack is full.
  • Underflow: Happens when trying to pop from an empty stack.
  • Tip: Always check for underflow before popping.

Operations on Stack

  • Push: Adds data at the top of the stack.
  • Pop: Removes the top data from the stack.
  • Peek: Looks at the top data without removing.
  • Tip: Always keep track of the top element for efficient operations.

Applications of Stack

  • Expression Evaluation: Stacks are used in converting and evaluating mathematical expressions.
  • Recursion: Function calls use stack memory.
  • Undo Feature: Applications like MS Word use stacks to track undo actions.
  • Tip: Multiple stacks can be used for memory management.

Queue Concepts

  • Queue Definition: Follows First In First Out (FIFO).
  • Example: Like a line at the bank – first person in is first served.
  • Enqueue: Adds an element to the rear.
  • Dequeue: Removes an element from the front.
  • Circular Queue: The queue wraps around to reuse spaces.
  • Tip: Circular queues avoid memory waste in static queues.

Queue Implementation

  • Enqueue: Adds an item to the rear of the queue.
  • Dequeue: Removes an item from the front of the queue.
  • Front: Points to the first element in the queue.
  • Rear: Points to the last element in the queue.
  • Tip: Always check if the queue is empty before dequeueing.

Circular Queue

  • Definition: A circular queue connects the end to the front to utilize space.
  • Operation: When adding an element, position calculated as (rear + 1) % size.
  • Tip: Helps in avoiding memory wastage and improves efficiency.

DeQueue (Double Ended Queue)

  • Definition: A DeQueue allows insertion and deletion at both ends.
  • Operations: EnqueueFront: Add at the front, EnqueueRear: Add at the rear.
  • Tip: Useful in scenarios where both ends need to be accessed frequently.

Multiple Stack in Queue

  • Concept: Utilize multiple stacks to implement a queue.
  • Operations: Push elements onto one stack for enqueue; pop from another for dequeue.
  • Tip: Helps to manage multiple queues efficiently using stacks.
